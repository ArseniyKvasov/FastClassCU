# FastClass

FastClass — сервис для создания интерактивных уроков по английскому языку.
Проект состоит из независимых сервисов, которые запускаются единым Docker
Compose-стеком.

## Сервисы

### Auth Service

Идентификация пользователей: гостевой доступ или полный вход через подключаемых
OAuth/OIDC-провайдеров. Сейчас настроены Yandex и VK; дополнительные провайдеры,
включая CU ID, добавляются конфигурацией.

### Frontend Service

Точка входа для браузера и BFF-слой: раздаёт статический frontend, хранит
сессию в HTTP-only cookie и проксирует запросы к внутренним API.

### Content Service

Хранит и управляет учебными материалами: уроками, их копиями, файлами и
операциями обработки загруженных материалов.

### Classroom Service

Управляет классами и составом учеников, а также интеграциями с внешними
whiteboard- и LiveKit-сервисами.

### Assignments Service

Назначает материалы ученикам и классам, отслеживает дедлайны и отправляет
уведомления через WebSocket.

### Answers Service

Хранит ответы учеников, черновики и статус сдачи, а также принимает снимки
совместного редактирования.

### Collaboration Service

Обеспечивает совместное редактирование письменных ответов в реальном времени
на базе Yjs/CRDT.

### AI Assistant Service

Создаёт учебный контент с помощью LLM: черновики уроков, подсказки по заданиям,
изображения и результаты AI-генераций.

### Analytics Service

Собирает аналитику пользовательской активности и качества материалов для
администраторов сайта.

## Быстрый запуск Docker

Требуются Docker Engine и Docker Compose v2.

```bash
cp .env.example .env
# Укажите OAuth-данные и замените development-секреты перед публичным запуском.
docker compose up --build -d
```

Compose автоматически создаёт JWT-ключи и запускает миграции до старта
приложений.

Проверить состояние и логи:

```bash
docker compose ps
docker compose logs -f
```

Остановить стек:

```bash
docker compose down
```

Удалить также данные локальных PostgreSQL:

```bash
docker compose down -v
```

По умолчанию frontend доступен на `http://localhost:8080`. Все host-порты,
OAuth-настройки, cookie, CORS, внешние интеграции и секреты задаются в
[`.env.example`](.env.example).
